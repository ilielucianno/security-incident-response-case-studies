## Key Incident: Crypto Phishing Attack

**What happened:**
- A financial analyst, under pressure to capitalize on a coin surge, was targeted by an attacker on Discord
- They were tricked into entering a wallet seed phrase into a fake "update" website
- I discovered the breach seconds after the seed was entered
- Through a real-time "tug of war" with the attacker, I recovered 90% of the funds

**Why it matters:**
- Demonstrated the importance of fast response (seconds matter)
- Showed how hardware decisions (faster systems) made a difference
- Highlighted that deep familiarity with tools is a security control
- Became a company-wide training case study

---

## Lessons Learned Across Incidents

| Lesson | Why It Matters |
|--------|----------------|
| **Seed phrase is NEVER shared** | Fundamental rule for crypto security |
| **Pressure creates mistakes** | Attackers create urgency to bypass logic |
| **Hardware matters** | Faster systems buy seconds that save money |
| **Physical presence helps** | Being in the office allows early detection |
| **Know your tools** | Familiarity = speed = better recovery |
| **Post-incident training** | Incidents become learning opportunities |
